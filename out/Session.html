Methods

control(jobId, action)

Hold, release, suspend, resume, or kill the job identified by jobId. If jobId is JOB_IDS_SESSION_ALL, then this routine acts on all jobs submitted during this DRMAA session up to the moment control() is called. The legal values for action are: - SUSPEND: stop the job, - RESUME: (re)start the job, - HOLD: put the job on-hold, - RELEASE: release the hold on the job, - TERMINATE: kill the job. This routine returns once the action has been acknowledged by the DRM system, but does not wait until the action has been completed.
Parameters:
Name Type Description
jobId number | string The id of the job to control, or the constant JOB_IDS_SESSION_ALL
action string The control action to be taken

synchronize(jobIds, timeoutnullable)

The synchronize() method returns when all jobs specified in jobIds have failed or finished execution. If jobIds contains JOB_IDS_SESSION_ALL, then this method waits for all jobs submitted during this DRMAA session. The caller may specify a timeout, indicating how many milliseconds to wait for this call to complete before timing out. The special value TIMEOUT_WAIT_FOREVER can be used to wait indefinitely for a result.
Parameters:
Name Type Attributes Description
jobIds Array.<number> | string The ids of the jobs to synchronize.

 The maximum number of milliseconds to wait for jobs' completion.
